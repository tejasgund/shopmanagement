🔗 API ENDPOINTS QUICK REFERENCE
================================================================================

1. GET /api/reports/pending-bills
   Headers: Authorization: Bearer <JWT_TOKEN>
   Params:  start_date, end_date, complex_id, user_id (all optional)
   Returns: JSON {success, period, filters, summary, records}

2. GET /api/reports/pending-bills/download
   Headers: Authorization: Bearer <JWT_TOKEN>
   Params:  start_date, end_date, complex_id, user_id (all optional)
   Returns: PDF file (application/pdf)

3. GET /api/reports/payments
   Headers: Authorization: Bearer <JWT_TOKEN>
   Params:  start_date, end_date, complex_id, user_id, payment_method (optional)
   Returns: JSON {success, period, filters, summary, records}

4. GET /api/reports/payments/download
   Headers: Authorization: Bearer <JWT_TOKEN>
   Params:  start_date, end_date, complex_id, user_id, payment_method (optional)
   Returns: PDF file (application/pdf)

All parameters are optional. Date format: ISO 8601 (YYYY-MM-DDTHH:MM:SS)

📊 FILTER COMBINATIONS (EXAMPLES)
================================================================================

Example 1: All pending bills
  /api/reports/pending-bills
  → No filters

Example 2: Pending bills for January 2026
  /api/reports/pending-bills?start_date=2026-01-01T00:00:00&end_date=2026-01-31T23:59:59
  → Date range filter

Example 3: Pending bills for complex 1
  /api/reports/pending-bills?complex_id=1
  → Complex filter

Example 4: Pending bills for user 5 in January 2026 from complex 1
  /api/reports/pending-bills?start_date=2026-01-01T00:00:00&end_date=2026-01-31T23:59:59&complex_id=1&user_id=5
  → Multiple filters combined

Example 5: Payment report by cash only
  /api/reports/payments?payment_method=Cash
  → Payment method filter

================================================================================
🆘 TROUBLESHOOTING
================================================================================

Issue: "Reports page shows blank"
  → Check browser console for errors
  → Verify JWT token exists in localStorage
  → Ensure backend is running

Issue: "Dropdowns are empty"
  → Check if logged in as admin
  → Verify database has complexes/users
  → Check network tab for failed requests

Issue: "PDF download fails"
  → Ensure backend is running
  → Check JWT token is valid
  → Try refreshing page

Issue: "No data displayed"
  → Check if data exists in database
  → Try removing date filters
  → Verify database connection

Issue: "API returns 401 Unauthorized"
  → Login again to get fresh token
  → Clear localStorage: localStorage.clear()
  → Refresh page
